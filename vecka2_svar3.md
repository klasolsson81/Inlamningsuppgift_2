# Vad innebär samarbete med GitHub? Vad är pull requests, branches och merge?

## Svar:
:octopus: Sammarbeten på GitHub innebär att flera personer kan arbeta med samma projekt/repos. 
Pull-request görs när man är klar med sina ändringar och vill "ladda upp" dom till GitHub.
En ansvarig för projektet/repot kan då kolla genom ändringarna och godkänna pull requiesten om allt ser bra ut.

:herb: Branches är olika grenar, vilket är bra när man jobbar med ändringar, utan att skada själva huvudbranchen.

:twisted_rightwards_arrows: Merge är när man slår ihop andra branches med huvudbranchen.
